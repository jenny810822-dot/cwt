# Frontend Design Skill Summary

This skill focuses on creating distinctive, production-grade frontend interfaces that prioritize design quality and avoid generic aesthetics.

## Key Principles

**Design Thinking**: Before coding, establish a clear aesthetic direction by understanding purpose, audience, tone, and technical constraints. The guidance emphasizes choosing "an extreme" visual approach—whether brutally minimal, maximalist, retro-futuristic, or another distinctive style—and executing it with intentionality.

**Critical Aesthetics Focus**:
- Typography: "Choose fonts that are beautiful, unique, and interesting. Avoid generic fonts like Arial and Inter"
- Color & Theme: Commit to cohesive palettes using CSS variables
- Motion: Prioritize high-impact animations over scattered micro-interactions
- Spatial Composition: Employ asymmetry, overlap, and grid-breaking elements
- Visual Details: Layer textures, gradients, and atmospheric effects

**What to Avoid**: Generic AI aesthetics including overused font families, clichéd color schemes (particularly purple gradients), predictable layouts, and "cookie-cutter design that lacks context-specific character."

The framework emphasizes that production-grade code must be "visually striking and memorable" while remaining functional, with implementation complexity matching the aesthetic vision—elaborate code for maximalist designs, restrained precision for minimalist approaches.
